# WingSync Site

This is the public WingSync website. It is a clean static HTML/CSS site suitable for GitHub Pages.

The site includes:

- `index.html` - WingSync overview
- `features.html` - Main app features
- `privacy-policy.html` - Privacy policy
- `terms.html` - Terms of use
- `support.html` - Support information
- `delete-account.html` - Delete account and data request information
- `style.css` - Shared styling

No secrets, Firebase keys, API keys, or app source code are included.

## Open Locally

Open `index.html` directly in a browser. The site uses plain static HTML and CSS, so no build step or local server is required.

## GitHub Pages Setup

1. Push this folder to a GitHub repository.
2. Open the repository on GitHub.
3. Go to **Settings**.
4. Open **Pages**.
5. Under **Build and deployment**, choose **Deploy from a branch**.
6. Select the branch to publish, usually `main`.
7. Select the root folder `/`.
8. Save the settings.

GitHub will publish the static site and show the public Pages URL after deployment finishes.

## Before Publishing

- Replace placeholder effective dates or publisher details if needed.
- Do not add app source code, Firebase keys, API keys, secrets, package configs, analytics scripts, trackers, fonts, CDNs, or third-party embeds.
- Host `privacy-policy.html` and `delete-account.html` publicly before submitting WingSync to app stores.
